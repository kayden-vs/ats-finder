# ats_finder

A standalone CLI tool that auto-detects which ATS (Applicant Tracking System) a
company uses, verifies the slug actually returns jobs, and outputs a valid
`companies.yaml` append-block for [jobradar](https://github.com/kayden-vs/jobradar)
plus a detailed CSV report.

---

## Supported ATS Platforms

| # | ATS | Detection method | Requires key? |
|---|-----|-----------------|---------------|
| 1 | **Greenhouse** | JSON API | No |
| 2 | **Lever** | JSON API | No |
| 3 | **Ashby** | JSON API | No |
| 4 | **Workable** | JSON API (POST) | No |
| 5 | **SmartRecruiters** | JSON API | No |
| 6 | **Rippling** | JSON API | No |
| 7 | **BambooHR** | JSON API | No |
| 8 | **Recruitee** | JSON API | No |
| 9 | **Personio** | XML feed | No |
| 10 | **Teamtailor** | HTML detection | Yes (flagged in output) |
| 11 | **Freshteam** | HTML detection | No (scrape-only) |

## Usage

```bash
# Basic run — uses companies.txt, writes companies_found.yaml + ats_report.csv
python main.py

# Full options
python main.py \
  --input  companies.txt \
  --output companies_found.yaml \
  --csv    ats_report.csv \
  --workers 15

# Skip specific ATS platforms (e.g. skip Workable's aggressive rate-limiting)
python main.py --skip-ats workable,teamtailor

# Preview what would be probed without making any requests
python main.py --dry-run

# Enable verbose/debug logging
python main.py --verbose
```

### CLI Arguments

| Argument | Default | Description |
|---|---|---|
| `--input`, `-i` | `companies.txt` | Newline-separated company names file |
| `--output`, `-o` | `companies_found.yaml` | YAML output path |
| `--csv`, `-c` | `ats_report.csv` | CSV report path |
| `--workers`, `-w` | `10` | Max concurrent companies processed at once |
| `--skip-ats` | _(none)_ | Comma-separated ATS names to skip |
| `--dry-run` | `false` | Print probe plan, make no requests |
| `--verbose`, `-v` | `false` | Enable DEBUG-level logging |

## Output: `ats_report.csv`

Full per-company probe report.

| Column | Values |
|---|---|
| `company_name` | Original name from input file |
| `detected_ats` | ATS platform name, or empty if not found |
| `slug_used` | The slug that produced a hit |
| `jobs_found` | Number of live jobs returned (-1 = unknown) |
| `careers_url` | Link to the company's careers page |
| `probe_time_ms` | Total probe time in milliseconds |
| `status` | `found`, `not_found`, `error`, `requires_key` |

## How Slug Generation Works

For each company name, `normalizer.py` generates multiple slug candidates in order:

```
"Razorpay India" →
  1. razorpay-india      (hyphenated lowercase)
  2. razorpayindia       (no separator lowercase)
  3. razorpay            (first word)
  4. RazorpayIndia       (PascalCase)
  5. Razorpay            (original first-word casing)
  6. razorpay            (stripped of common suffixes like "india")
```

Each ATS prober tries all candidates in order and stops at the first verified hit.

## Architecture

```
main.py
  └─ asyncio.run(run())
       ├─ httpx.AsyncClient (HTTP/2 enabled, global semaphore caps concurrency)
       ├─ For each company:
       │    ├─ normalizer.generate_slugs()  → slug candidates
       │    └─ detector.probe_all()
       │         ├─ probe_greenhouse()  ─┐
       │         ├─ probe_lever()        │  all run concurrently
       │         ├─ probe_ashby()        │  via asyncio.gather()
       │         ├─ probe_workable()    ─┘  (Workable serialized + 3s sleep)
       │         ├─ probe_smartrecruiters()
       │         ├─ probe_rippling()
       │         ├─ probe_bamboohr()
       │         ├─ probe_recruitee()
       │         ├─ probe_personio()
       │         ├─ probe_teamtailor()
       │         └─ probe_freshteam()
       └─ output.checkpoint()  every 10 companies  →  .yaml + .csv
```

### Concurrency & Rate Limiting

- **Global semaphore**: caps simultaneous HTTP requests (`--workers` flag).
- **Per-ATS domain delay**: 100ms between requests to the same domain.
- **Workable**: serialized via an `asyncio.Lock` + mandatory 3s sleep after each probe.
- **BambooHR**: `follow_redirects=False` — a redirect means the company isn't hosted there.
- **429 handling**: exponential backoff (`2^attempt` seconds, max 3 retries).
- **Incremental saves**: every 10 companies, results are flushed to disk — a crash won't lose all progress.

## Integrating with jobradar

After running `ats_finder`, append the YAML output to your existing `companies.yaml`:

```bash
cat companies_found.yaml >> ../jobradar/companies.yaml
```

Or paste the relevant sections manually — the format exactly matches jobradar's schema.

---

## Notes & Caveats

- **Workable rate-limits aggressively** (~100 requests / IP). Use `--skip-ats workable`
  if you're running the tool repeatedly in a short window.
- **Teamtailor** entries in the YAML output are comments only — you'll need to obtain a
  per-company API key from Teamtailor and add those entries manually.
- **Freshteam** is detected via HTML only. There is no public JSON API; add these companies
  to a custom scraper if needed.
- **SmartRecruiters slugs are case-sensitive** and inconsistently formatted. The prober
  tries multiple casing variants automatically.
- **Personio** tries both `.de` and `.com` TLDs.
- All errors are caught per-company — a network blip won't abort the entire run.
